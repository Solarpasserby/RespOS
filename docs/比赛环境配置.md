## 目标

搭建 OS 比赛项目所需的开发环境：拉取赛题组提供的 Docker 镜像，在 WSL 中克隆并存放项目源码，通过 VS Code 的 Dev Containers 功能进入容器进行开发、编译与调试。

本教程采用的开发环境组织方式如下：

- **Windows**：运行 VS Code，作为主要编辑与管理界面
- **WSL**：存放项目源码，并提供日常 Linux 命令行环境
- **Docker 容器**：提供比赛要求的统一工具链和隔离的开发环境
- **Dev Containers**：将 VS Code 与容器连接起来，实现“在容器中开发”

赛题组提供的信息如下：

- Dockerfile 与工具链说明：`https://github.com/zhouzhouyi-hub/os-contest-image/`
- 比赛镜像：`zhouzhouyi/os-contest:20260104`

### 采用这种方案的原因

1. **工具链一致**
    使用比赛镜像可以尽量保证本地开发环境与赛题组提供的环境一致，减少由于编译器、依赖版本或系统差异带来的问题。
2. **环境隔离**
    容器能够将比赛开发环境与本机系统隔离开，避免本机配置被污染，也便于后续重建和迁移。
3. **源码放在 WSL 中更合适**
    将源码放在 WSL 的 Linux 文件系统中，通常比放在 `D:\`、`C:\` 这类 Windows 挂载盘中更适合 Linux 开发场景；对于 Git、Rust/Cargo、文件监听、语言服务器等涉及大量小文件读写的操作，性能和兼容性通常都会更好。

## 安装 WSL2 并克隆项目文件

> 本节部分参考[此篇博客](https://blog.csdn.net/Natsuago/article/details/145594631)，可以配合食用

### 整体思路

**启用 WSL2** -> **正常安装 Ubuntu** -> **把 Ubuntu 迁移到 D 盘**

### 整体流程

#### 1. 直接安装 WSL

以**管理员身份**打开 PowerShell。执行如下命令，这会安装 WSL、启用所需组件，并安装 Ubuntu 24.04 发行版

```shell
wsl --install -d Ubuntu-24.04
```

显式设置默认 WSL 版本为 2

```shell
wsl --set-default-version 2
```

#### 2. 首次启动 Ubuntu，完成初始用户创建

安装完成后，启动 Ubuntu。第一次启动时，系统通常会提示你创建。这里正常创建即可

- Linux 用户名
- Linux 密码

创建完成后，建议进入 Ubuntu 执行一次更新。这一步不是 WSL 必需步骤，但很推荐，能减少后续开发环境问题

```bash
sudo apt update
sudo apt upgrade -y
```

#### 3. 导出当前 Ubuntu

假设你准备把 Ubuntu 放到 `D:\WSL\Ubuntu-24.04` ，先在 D 盘手动建好这个目录

在 PowerShell 执行，查看当前安装的发行版名称和对应的 WSL 版本

```shell
wsl -l -v
```

你大概率会看到类似这样的输出，记住你的发行版名称，后面要考

```shell
  NAME            STATE           VERSION
* Ubuntu-24.04    Running         2
```

执行以下命令，这会把当前发行版导出成一个 tar 包，相当于做一份可迁移备份

```shell
wsl --export Ubuntu-24.04 D:\WSL\Ubuntu-24.04\ubuntu.tar
```

**确认 tar 已导出成功后**，执行以下命令，这会把原先注册在 WSL 中的 Ubuntu 删除

```shell
wsl --unregister Ubuntu-24.04
```

#### 4. 重新导入到 D 盘

执行以下命令

```shell
wsl --import Ubuntu-24.04 D:\WSL\Ubuntu-24.04 D:\WSL\Ubuntu-24.04\ubuntu.tar --version 2
```

这个命令的含义是：

- `Ubuntu-24.04`：导入后的发行版名称
- `D:\WSL\Ubuntu-24.04`：安装位置
- `ubuntu.tar`：导入源文件
- `--version 2`：指定为 WSL2

执行以下命令检查结果，如果看到发行版还在，而且版本是 2，就说明迁移成功了

```shell
wsl -l -v
```

#### 5. 配置普通用户（非必要）

通过 `wsl --import` 导入的发行版，第一次启动时**往往默认进入 root**，并不会自动帮你创建普通用户。所以导入后你通常需要手动创建普通用户。先启动它：

```shell
wsl -d Ubuntu-24.04
```

此时你就进入了 Ubuntu，然后执行以下命令。这样就创建了普通用户，并给了 sudo 权限

```bash
adduser yourname
usermod -aG sudo yourname
```

在 Ubuntu 中修改 `/etc/wsl.conf` 文件

```bash
sudo nano /etc/wsl.conf
```

写入以下数据以设置默认用户为普通用户

```bash
[user]
default=yourname
```

保存后，输入指令退出 Ubuntu

```bash
logout
```

在 Windows PowerShell 执行：

```shell
wsl --shutdown
```

然后在重新打开 Ubuntu 便完成了 WSL2 的安装

## 安装 Docker Desktop 并下载比赛镜像

完成 WSL 与 Ubuntu-24.04 的安装后，下一步需要安装 Docker Desktop，并将其与 WSL 中的 Ubuntu 发行版打通，最后下载比赛所需镜像

### 1. 安装 Docker Desktop

前往 [Docker 官网](https://www.docker.com/) 下载并安装 Docker Desktop。安装过程中一般保持默认选项即可。Docker 的使用一般需要科学上网

### 2. 配置镜像存储位置

安装完成并登录 Docker Desktop 后，进入设置页面

在 `Settings -> Resources -> Advanced` 中，可以修改 Docker 镜像、容器等数据的存放位置

> 镜像默认放在 C 盘，而你希望减少 C 盘占用，建议在这里提前改到空间更充足的磁盘位置

### 3. 开启 WSL 集成

为了让 WSL 中的 Ubuntu-24.04 可以直接使用 Docker，还需要在 Docker Desktop 中开启 WSL 集成

进入 `Settings -> Resources -> WSL Integration` 并勾选 "Enable integration with additional distros:" 下面我们创建的 Ubuntu-24.04

### 4. 下载比赛镜像

回到 Docker Desktop 主界面，在搜索栏中输入 "zhouzhouyi/os-contest:20260104"，找到对应镜像后执行下载。下载完成后，就可以在本地看到该比赛镜像，后续会基于这个镜像创建容器

## 在 VS Code 中使用 Dev Container 插件打开项目

完成 Docker Desktop、WSL 和比赛镜像的准备后，可以在 VS Code 中通过 Dev Container 打开项目

### 1. 安装 Dev Containers 插件

先在 Windows 上安装并打开 VS Code，然后进入扩展商店，搜索并安装 "Dev Containers"

### 2. 在容器中打开项目

首先需要使用 VS Code 打开你的项目，这一目标可以通过两个途径完成

- A. 在终端中进入 WSL 下的项目目录，随后输入 `code .`
- B. 依赖 VS Code 的远程连接插件连接 wsl，并打开指定目录

最后在 VS Code 中按下 `Ctrl + Shift + P` 打开命令面板，输入 `Dev Containers: Reopen in Container` 并选择**在容器中重新打开**
