## 目标

搭建 RespOS 初赛设计文档的撰写环境：安装 Typst 排版引擎和中文字体，实现一条命令编译 PDF，并在 VS Code 中实时预览。

## 工具链概览

| 工具 | 版本 | 用途 |
|------|------|------|
| Typst | 0.11.1 | 排版引擎，将 `.typ` 源文件编译为 PDF |
| 文鼎中文字体 | — | 提供宋体、楷体、明朝体等中文渲染 |
| VS Code + PDF 插件 | — | 编辑源文件 + 预览 PDF |

跟 Chronix 使用相同的 Typst 版本，无需 LaTeX、pandoc 等重型工具。

## 安装 Typst

Typst 提供预编译二进制，下载解压即可使用，无需编译安装。

```bash
# 下载并解压
wget -q https://github.com/typst/typst/releases/download/v0.11.1/typst-x86_64-unknown-linux-musl.tar.xz
tar -xf typst-x86_64-unknown-linux-musl.tar.xz

# 安装到用户目录
mkdir -p ~/.local/bin
mv typst-x86_64-unknown-linux-musl/typst ~/.local/bin/

# 加入 PATH（如果还没加过）
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

# 验证
typst --version
# 输出: typst 0.11.1 (786457e3)
```

> 如果宿主机是 WSL，可以跳过字体安装——`build.sh` 会自动引用 Windows 字体目录（`/mnt/c/Windows/Fonts`），直接用系统自带的 SimSun、SimHei、KaiTi 等字体。

## 安装中文字体（Dev Container / 纯 Linux 环境）

项目 `docs/初赛文档/fonts/` 目录已自带四种文鼎中文字体，直接复制到用户字体目录即可：

```bash
mkdir -p ~/.fonts
cp docs/初赛文档/fonts/*.ttf docs/初赛文档/fonts/*.ttc ~/.fonts/
fc-cache -f ~/.fonts/

# 验证
fc-list :lang=zh
```

> `build.sh` 已配置 `--font-path fonts`，如果不想复制到系统，编译时也会自动加载项目内字体。

### 字体说明

| 字体文件 | 名称 | 风格 | 文档中用途 |
|----------|------|------|------------|
| `gbsn00lp.ttf` | AR PL SungtiL GB（文鼎简报宋） | 宋体 | 正文默认字体 |
| `gkai00mp.ttf` | AR PL KaitiM GB（文鼎简中楷） | 楷体 | 封面校名 |
| `uming.ttc` | AR PL UMing CN（文鼎明体） | 明朝体 | 章节标题 |
| `ukai.ttc` | AR PL UKai CN（文鼎行楷） | 行楷 | 备用 |

Typst 的字体回退机制为：优先使用 Dev Container 中的文鼎字体，如果找不到则回退到 Windows 系统字体（SimSun、SimHei 等）。字体配置在 `main.typ` 文件头部的 `#let font-*` 变量中定义，后续如需更换字体可直接修改。

## 安装 VS Code PDF 预览插件

在 VS Code 扩展商店（Ctrl+Shift+X）搜索 `vscode-pdf`（作者 tomoki1207），点击 Install。

安装后，在 VS Code 中点击 `docs/初赛文档/初赛文档.pdf` 即可直接在编辑器内预览。

## 日常使用

### 工作流

```
编辑 main.typ  →  bash build.sh  →  点击 PDF 预览
     ↑                ↑                ↑
  VS Code 里改      终端执行      侧边栏打开 PDF
```

### 编译命令

```bash
bash docs/初赛文档/build.sh
```

`build.sh` 会自动检测运行环境：
- **WSL**：自动添加 `--font-path /mnt/c/Windows/Fonts` 引用 Windows 字体
- **Dev Container / Linux**：使用系统安装的文鼎字体

### 源文件说明

| 文件 | 作用 | 是否需要编辑 |
|------|------|--------------|
| `main.typ` | 文档源文件，包含样式设定和全部章节内容 | 是，写内容时编辑这个 |
| `build.sh` | 一键编译脚本 | 否 |
| `sdu-logo.svg` | 山东大学 logo 矢量图 | 否 |
| `初赛文档.pdf` | 编译产物 | 否，只读 |

## Typst 速查

Typst 语法类似 Markdown，常用操作：

```typst
= 一级标题
== 二级标题
=== 三级标题

// 正文直接写，段落间空一行

// 代码块
```rs
pub fn main() {
    println!("Hello");
}
```

// 表格
#table(
  columns: (auto, auto),
  [表头1], [表头2],
  [内容1], [内容2],
)

// 图片
#figure(caption: [架构图])[#image("arch.png", width: 80%)]

// 加粗
*加粗文字*

// 引用
TODO: 参考文献
```

更多语法参考官方文档：https://typst.app/docs
